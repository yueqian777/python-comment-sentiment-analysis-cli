# 网络评论情感分析系统学习说明

## 1. 这个程序解决什么问题

这个程序读取一份评论 CSV 文件，对每条评论判断情感倾向，结果分为三类：

- `positive`：正面评论
- `negative`：负面评论
- `neutral`：中性评论

同时，程序还会统计评论中的高频关键词，并输出情感分类的数量和占比。

## 2. 程序运行流程

整体流程很直接：

1. 从 CSV 文件读取评论文本。
2. 对评论做文本清洗，去掉网址、标点和多余空格。
3. 使用 `jieba` 对中文评论进行分词。
4. 用正面词和负面词给评论打分。
5. 根据得分判断评论是正面、负面还是中性。
6. 汇总所有评论的分类结果。
7. 输出 CSV、文本报告和柱状图。

对应到代码里，主要在 `src/sentiment_cli/analyzer.py`：

- `clean_text()`：清洗文本
- `tokenize()`：分词并去掉停用词
- `classify_text()`：判断单条评论情感
- `analyze_comments()`：批量分析评论
- `top_keywords()`：统计高频关键词
- `sentiment_summary()`：统计情感分类数量和占比

命令行入口在 `src/sentiment_cli/main.py`，它负责接收参数、读取 CSV、调用分析函数、保存结果。

## 3. 文本清洗用了什么知识

文本清洗主要用了正则表达式。

程序会把评论里的网址、标点符号、特殊符号处理掉，只保留中文、英文和数字。这样后面分词和统计时不会被无关字符干扰。

需要掌握：

- 字符串基本处理
- `re.sub()` 的用法
- 空白字符合并
- 函数封装

## 4. 分词用了什么知识

中文句子不像英文一样天然用空格分词，所以程序使用 `jieba`：

```python
jieba.lcut("物流速度很快")
```

它会把一句话切成多个词。程序还加入了停用词过滤，比如“的”“了”“很”这类词一般不适合作为关键词。

需要掌握：

- `jieba.lcut()`
- 列表遍历
- 停用词过滤
- `set` 查询速度快的特点

## 5. 情感分类逻辑

默认分类方式是词典打分法，不复杂，适合课程作业理解。

程序准备了两组词：

- 正面词：如“很好”“不错”“满意”“推荐”
- 负面词：如“差”“失望”“难吃”“糟糕”

分析一条评论时：

1. 看评论中出现了多少正面词。
2. 看评论中出现了多少负面词。
3. 正面词更多就是 `positive`。
4. 负面词更多就是 `negative`。
5. 两边一样或都没有就是 `neutral`。

这种方法的优点是容易理解、容易调试。缺点是对复杂语气不够敏感，比如反讽、双重否定、长句上下文等。

## 6. 关键词统计逻辑

关键词统计使用了 Python 标准库里的 `Counter`。

程序先把所有评论分词，然后统计每个词出现了多少次，最后取出现次数最多的前几个词。

需要掌握：

- `collections.Counter`
- 列表和循环
- 元组列表，如 `[("手机", 3), ("物流", 2)]`

## 7. pandas 在程序中的作用

`pandas` 主要用于读写 CSV。

程序用：

```python
pd.read_csv(...)
```

读取评论数据，用：

```python
result.to_csv(...)
```

保存分析后的结果。

需要掌握：

- `DataFrame`
- CSV 文件读取和保存
- 按列取数据
- 表格数据和普通列表之间的关系

## 8. sklearn 示例做了什么

`src/sentiment_cli/ml_model.py` 里放了一个机器学习版本的示例。

它使用：

- `TfidfVectorizer`：把文本转换成数字特征
- `LogisticRegression`：训练分类模型
- `Pipeline`：把特征提取和模型训练连起来

这个部分是提升内容。默认命令行程序使用词典法，因为它更容易解释；如果以后有大量已经标注好的评论数据，就可以使用 `sklearn` 模型提高分类效果。

需要掌握：

- 什么是训练数据
- 什么是标签
- 文本如何变成数字特征
- `fit()` 和 `predict()` 的区别
- 机器学习模型需要足够多的数据

- `assert` 判断结果是否符合预期

## 11. 你应该重点掌握什么

建议按这个顺序学习：

1. 会读懂 `main.py`，知道命令行参数怎么传进程序。
2. 会读懂 `analyzer.py`，知道每个函数负责哪一步。
3. 会修改正面词、负面词和停用词。
4. 会换一份自己的 CSV 数据运行程序。
5. 会查看 `classified_comments.csv` 和 `summary.txt`。
6. 会运行测试，并理解测试失败时说明什么。
7. 理解 `ml_model.py` 只是机器学习提升示例，不是默认主流程。

## 12. 可以继续改进的方向

- 增加更多正面词、负面词和停用词。
- 使用更大的人工标注评论数据训练 sklearn 模型。
- 增加混淆矩阵、准确率、召回率等模型评估指标。
- 改成 Streamlit 网页版，支持上传 CSV 文件。
- 增加评论来源字段，比如商品、电影、外卖平台。
