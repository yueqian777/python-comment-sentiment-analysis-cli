# 网络评论情感分析系统学习说明

## 1. 这个程序解决什么问题

这个程序读取一份评论 CSV 文件，对每条评论判断情感倾向，结果分为三类：

- `positive`：正面评论
- `negative`：负面评论
- `neutral`：中性评论

同时，程序还会统计评论中的高频关键词，并输出情感分类的数量和占比。

## 2. 程序运行流程

整体流程很直接：

1. 从 CSV 文件读取评论文本。
2. 对评论做文本清洗，去掉网址、标点和多余空格。
3. 使用 `jieba` 对中文评论进行分词。
4. 用正面词和负面词给评论打分。
5. 根据得分判断评论是正面、负面还是中性。
6. 汇总所有评论的分类结果。
7. 输出 CSV、文本报告、数量图和占比图。

对应到代码里，主要在 `src/sentiment_cli/analyzer.py`：

- `clean_text()`：清洗文本
- `tokenize()`：分词并去掉停用词
- `classify_text()`：判断单条评论情感
- `analyze_comments()`：批量分析评论
- `top_keywords()`：统计高频关键词
- `sentiment_summary()`：统计情感分类数量和占比
- `save_sentiment_count_chart()`：保存三类评论的数量柱状图
- `save_sentiment_ratio_chart()`：保存直接标出百分比的占比图

命令行入口在 `src/sentiment_cli/main.py`，它负责接收参数、读取 CSV、调用分析函数、保存结果。

## 3. 文本清洗用了什么知识

文本清洗主要用了正则表达式。

程序会把评论里的网址、标点符号、特殊符号处理掉，只保留中文、英文和数字。这样后面分词和统计时不会被无关字符干扰。

需要掌握：

- 字符串基本处理
- `re.sub()` 的用法
- 空白字符合并
- 函数封装

## 4. 分词用了什么知识

中文句子不像英文一样天然用空格分词，所以程序使用 `jieba`：

```python
jieba.lcut("物流速度很快")
```

它会把一句话切成多个词。程序还加入了停用词过滤，比如“的”“了”“很”这类词一般不适合作为关键词。

需要掌握：

- `jieba.lcut()`
- 列表遍历
- 停用词过滤
- `set` 查询速度快的特点

## 5. 情感分类逻辑

默认分类方式是词典打分法，不复杂，适合课程作业理解。

程序准备了两组词：

- 正面词：如“很好”“不错”“满意”“推荐”
- 负面词：如“差”“失望”“难吃”“糟糕”

分析一条评论时：

1. 看评论中出现了多少正面词。
2. 看评论中出现了多少负面词。
3. 正面词更多就是 `positive`。
4. 负面词更多就是 `negative`。
5. 两边一样或都没有就是 `neutral`。

为了减少中文否定词误判，程序还会检查情感词前 1 到 3 个中文字符。如果附近出现“不”“没”“没有”“不是”“无”“并不”，就把这个情感词的倾向反转。例如，“不满意”中的“满意”会按负面处理，“不贵”中的“贵”会按正面处理。

这种方法的优点是容易理解、容易调试。缺点是规则仍然比较简单，对反讽、双重否定、长句上下文等复杂语气不够敏感。

## 6. 关键词统计逻辑

关键词统计使用了 Python 标准库里的 `Counter`。

程序先把所有评论分词，然后统计每个词出现了多少次，最后取出现次数最多的前几个词。

需要掌握：

- `collections.Counter`
- 列表和循环
- 元组列表，如 `[("手机", 3), ("物流", 2)]`

`pandas` 主要用于读写 CSV。

程序用：


## 8. sklearn 评估流程

`src/sentiment_cli/ml_model.py` 使用 TF-IDF + LogisticRegression 训练机器学习模型，`src/sentiment_cli/evaluate.py` 负责完成评估闭环。

评估流程是：

1. 读取包含 `comment` 和 `label` 的标注 CSV。
2. 使用 `train_test_split` 划分训练集和测试集。
3. 优先按标签做 `stratify` 分层抽样；类别样本过少时退回普通随机划分。
4. 只用训练集训练机器学习模型。
5. 在同一个测试集上分别运行词典法和机器学习法。
6. 输出准确率、分类报告和混淆矩阵，方便直接比较。

默认基础分析仍使用词典法，因为它更容易解释；机器学习法是否更准确，需要通过足够多、质量可靠的标注数据来验证。

需要掌握：

- `TfidfVectorizer` 如何把文本转换成数字特征
- `LogisticRegression` 如何训练分类模型
- `fit()` 和 `predict()` 的区别
- 训练集和测试集必须分开
- 两种方法要在同一个测试集上比较
- 机器学习模型需要足够多的数据

## 9. 情感数量图和占比图

程序用 `matplotlib` 生成两张图：

```text
outputs/sentiment_count.png
outputs/sentiment_ratio.png
```

`sentiment_count.png` 用柱状图展示三类评论各有多少条。`sentiment_ratio.png` 用百分比柱状图展示三类评论各占总数的多少，并把百分比直接写在柱子上。这样“数量”和“占比”不会混在一起。

需要掌握：

- 柱状图 `bar`
- 设置标题和坐标轴
- 在柱子上标注数值或百分比
- 保存图片 `savefig`

## 10. accuracy 和混淆矩阵

`accuracy` 表示预测正确的样本数占测试集总样本数的比例。例如测试集有 10 条评论，其中 8 条预测正确，accuracy 就是 0.8。

混淆矩阵会把“真实标签”和“预测标签”交叉统计。主对角线上的数字表示预测正确的数量，其他位置表示某一类被错分成另一类的数量。它比只看 accuracy 更具体，可以发现模型是否经常把 neutral 误判成 positive 或 negative。

6. 会运行测试，并理解测试失败时说明什么。

## 13. 可以继续改进的方向

- 增加更多正面词、负面词和停用词。
- 使用更大的人工标注评论数据训练 sklearn 模型。
- 收集更多真实标注评论，并单独保留最终测试集。
- 调整 TF-IDF 和逻辑回归参数，比较不同模型效果。
- 改成 Streamlit 网页版，支持上传 CSV 文件。
- 增加评论来源字段，比如商品、电影、外卖平台。

这个项目当前先保持命令行版，代码结构比较平，后面要改网页或接口版时，可以继续复用 `analyzer.py` 和 `evaluate.py` 里的核心逻辑。当前数据规模仍是课程作业演示级别，评估结果不能直接代表真实平台效果。
